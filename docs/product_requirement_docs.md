# 製品要求仕様書 (PRD)

## 1. 導入

### 1.1 プロジェクト名

image-annotator-lib

### 1.2 プロジェクト目標

- 多様な画像アノテーションタスク（タギング、スコアリングなど）のための再利用可能なPythonライブラリを提供する。
- 異なるアノテーションタイプやモデルに対応する、柔軟で拡張可能なアーキテクチャを提供する。これには、ローカルMLモデルと、**Google、OpenAI、Anthropicなどの主要な外部Web API**が含まれる。
- **`pydantic-ai` を活用し、Web APIから構造化された信頼性の高いアノテーションデータを取得する。**
- **プロバイダーレベルでのリソース共有（`ProviderManager`）により、Web API利用の効率性を最大化する。**
- 他のアプリケーションやワークフローとの容易な統合を実現する。
- 高いコード品質、テストカバレッジ、そして明確なドキュメントを維持する。
- 統一されたAPIを提供する。

### 1.3 対象ユーザー

- 画像アノテーション機能が必要なアプリケーション開発者。
- アノテーション付き画像データセットを扱う研究者。
- 機械学習モデル用の画像データを準備するデータサイエンティスト。

## 2. 機能

### 2.1 主要機能

- **統一されたアノテーションインターフェース**: `annotate` 関数を通じて、ローカルモデルとWeb APIモデルを透過的に扱う。
- **多様なモデルサポート**:
    - **ローカルモデル**: ONNX形式などのモデルを効率的にロードし、推論を実行（`ModelLoad`）。
    - **Web APIモデル**: Google (Gemini), OpenAI (GPT), Anthropic (Claude) などの主要なLLM/VLMをサポート。
- **効率的なリソース管理**:
    - **`ProviderManager`**: Web APIプロバイダーごとにクライアントインスタンスを共有し、接続のオーバーヘッドを削減。
    - **`PydanticAIProviderFactory`**: `pydantic-ai` の `Agent` をキャッシュし、再利用することでリソース効率を向上。
- **構造化出力**: `pydantic-ai` と `AnnotationSchema` を利用し��、Web APIからの出力を安定したJSON形式に整形。
- **拡張性**:
    - 新しいモデルやAPIプロバイダーを容易に追加できるプラグイン的なアーキテクチャ（`ModelRegistry`とクラス継承）。
    - 設定ファイル（`models.toml`）による柔軟なモデル構成管理。
- **堅牢なエラーハンドリング**: APIエラーやメモリ不足など、一般的なエラーを適切に処理。

## 3. プロジェクト概要

`image-annotator-lib` は、`scorer-wrapper-lib` と `tagger-wrapper-lib` を統合し、ローカルの機械学習モデルや外部のWeb APIを利用して、画像アノテーション機能を提供するPythonライブラリである。統一されたAPIを通じて多様なアノテーションソースを透過的に扱うことを目指しており、特に `ProviderManager` と `pydantic-ai` の活用により、効率的で信頼性の高いWeb API連携を実現している。

# 変更履歴

## 2025-06-25: Provider-level管理アーキテクチャの導入

- **目的**: Web APIを利用するモデルの効率性、拡張性、保守性を向上させる。
- **変更点**:
    - **`ProviderManager`の導入**: APIプロバイダー（Google, OpenAIなど）ごとに単一のクライアントインスタンスを管理・共有する仕組みを導入。これにより、リクエストごとの接続オーバーヘッドを削減。
    - **`PydanticAIProviderFactory`の導入**: `pydantic-ai` の `Agent` をプロバイダーレベルでキャッシュ・再利用するファクトリを実装。リソースの効率的な利用を促進。
    - **`PydanticAIAnnotatorMixin`の導入**: `pydantic-ai` を利用するアノテータの共通ロジックをミックスインとして抽出し、コードの重複を削減。
- **ユーザー価値**:
    - **パフォーマンス向上**: APIクライアントの再利用により、Web APIへのリクエスト処理が高速化。
    - **保守性の向上**: プロバイダーごとの関心事が分離され、新しいAPIの追加や既存APIの仕様変更が容易に。
    - **信頼性の向上**: `pydantic-ai` による構造化出力の強制により、APIからのレスポンス形式が安定。

## 2025-05-10: Web APIアノテータの責務分離と型定義統一

- **目的**: 保守性と拡張性を高めるため、APIごとの実装を明確に分離。
- **変更点**:
    - `OpenAIApiAnnotator` と `AnthropicApiAnnotator` を個別のファイルに分離。
    - `AnnotationSchema` を用いて、構造化出力の型仕様を統一。
    - 各APIに対応するユニットテストを強化。
- **ユーザー価値**:
    - APIごとの責務が明確になり、開発・運用の効率が向上。
    - 新機能の追加やデバッグが容易に。
